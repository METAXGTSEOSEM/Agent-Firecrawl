## v2.11.0 is live

Firecrawl v2.11.0 is our first release since `v2.10` on May 15 and consolidates five weeks of work into a single drop. It lands PII redaction directly on scrape, ships a `deterministicJson` format that codegens reusable extractors instead of calling an LLM per request, adds generic video discovery to the `video` format, and makes the Monitor API smarter with goal-driven LLM judges, JSON-mode diffs, and snapshots. It also exposes `createdAt` / `completedAt` / `duration` on crawl status, splits the `searchZDR` forced mode into explicit ZDR and anonymous variants, migrates self-host deployments from the Supabase SDK to Drizzle on native Postgres, adds a typed `/v2/search/research` proxy with cross-language SDKs, makes `/interact` a full alias for `/v2/browser` and surfaces the raw CDP WebSocket URL on interact responses, opens `/v2/parse` to keyless callers with atomic per-IP credit reservation, and patches a fresh batch of CVEs.

### Highlights

- **`redactPII` option** — Set `redactPII: true` on `/v2/scrape`, `/v2/batch/scrape`, `/v2/crawl`, `/v2/parse`, or `/v2/extract` and `document.markdown` comes back with PII redacted. The object form accepts `mode` (`accurate` | `aggressive` | `fast`), an `entities[]` filter, and `replaceStyle` (`tag` | `mask` | `remove`). Costs +4 credits per scrape (plus +4 per extra PDF page) and is exposed across the JS, Python, Java, .NET, Go, PHP, Ruby, Rust, and Elixir SDKs.
- **`deterministicJson` format** — A new scrape format that codegens a reusable JS extractor for your schema and runs it in a sandbox instead of calling an LLM on every request. Populates `document.json` like the existing `json` format, bills 10 credits per page when this run generates a fresh extractor and 3 credits per page on cached reuse, and gets faster the more you reuse the same schema.
- **Generic video discovery** — The `video` format now returns a structured `document.videos` array with direct video URLs, HLS manifests, embeds, thumbnails, titles, durations, and provider metadata extracted from page HTML, alongside the existing `document.video` string for supported providers like YouTube.
- **Crawl wall-clock timestamps** — `GET /v2/crawl/{id}` and `GET /v2/batch/scrape/{id}` now return `createdAt`, `completedAt`, and `duration` so callers can read actual crawl wall-clock time directly from the API.
- **`/v2/search/research` proxy** — New typed research endpoints at `GET /v2/search/research/papers`, `/papers/:id`, `/papers/:id/similar`, and `/github`, billed against `SEARCH_CREDITS`. The legacy `/v2/research/*` mount is kept as a deprecated alias. Available across the JS, Python, Java, .NET, Go, PHP, Ruby, Rust, and Elixir SDKs.
- **`/interact` is a `/browser` alias** — `POST /interact`, `GET /interact`, `POST /interact/:sessionId/execute`, and `DELETE /interact/:sessionId` are now full aliases for `/v2/browser` with identical behavior, rate limits, and the 2-credit charge on session create.
- **`cdpUrl` on interact responses** — `POST /v2/scrape/:jobId/interact` and `/v2/browser` execute responses now include the raw CDP WebSocket URL alongside the iframe view URLs, so you can attach Playwright, Puppeteer, or any CDP client directly to the live browser session. Surfaced as `cdpUrl` (JS) and `cdp_url` (Python).
- **Generic `/v2/feedback` endpoint** — A new `POST /v2/feedback` route covers search, scrape, parse, and map jobs with shared recording and refund logic and per-endpoint refund-policy snapshots; the legacy `/v2/search/:jobId/feedback` route keeps working.
- **Monitor goals with LLM judge** — Set `goal` and `judgeEnabled` on a monitor and every changed page is classified as meaningful vs noise, sorted into summary emails with the unified diff rendered inline. Typed in the JS, Python, Java, .NET, Go, PHP, Ruby, Rust, and Elixir SDKs.
- **Monitor JSON diffs and snapshots** — Monitors using `{type: "json"}` or `{type: "changeTracking", modes: ["json"]}` now diff structured fields against the previous run and persist a `snapshot.json` of current values, surfaced through `GET /v2/monitor/:id/checks/:checkId`.
- **Mixed-mode change tracking** — Requesting both `["json", "git-diff"]` modes now runs both diffs and reports `changed` if either side changes, with the markdown unified diff attached as a sidecar on the JSON artifact.
- **Monitor email confirmation** — External monitor recipients now confirm opt-in before they receive alerts and get a one-click unsubscribe link in every summary email; team members are auto-confirmed.
- **Dedicated search credits** — `/v2/search` now meters against its own `SEARCH_CREDITS` pool rather than the shared scrape/crawl/extract `CREDITS` pool. `/v2/team/credit-usage` reports both.
- **Agent discovery on `401`** — Unauthorized responses now include `WWW-Authenticate: Bearer realm="firecrawl"` so MCP and agent clients can discover the credential scheme programmatically.
- **`forced-zdr` and `forced-anon` search modes** — `searchZDR: "forced"` has been split into `"forced-zdr"` and `"forced-anon"` to disambiguate ZDR enterprise routing from anonymous routing; `"forced"` continues to work as a deprecated alias.
- **Self-host Drizzle migration** — The API now runs on Drizzle ORM against native Postgres. Self-host deployments switch from `SUPABASE_*` env vars to `DATABASE_URL`, optional `DATABASE_REPLICA_URL`, and `INDEX_DATABASE_URL`; Stripe auto-recharge is removed in favor of Autumn-driven credit top-ups.
- **SemVer Docker image tags** — Production API images now publish to `ghcr.io/firecrawl/firecrawl` as immutable `X.Y.Z` SemVer manifests plus `X.Y`, `X`, and `latest` aliases. The legacy `vX.Y.Z` Docker tag is no longer published — pull `X.Y.Z` (or `X.Y` / `X` / `latest`) instead.
- **Keyless `/v2/parse` and atomic credit reservation** — `POST /v2/parse` now accepts keyless callers, and keyless requests across `/v1` and `/v2` scrape, search, parse, and interact atomically reserve their projected credits before work starts so concurrent multi-credit calls can no longer race past the daily cap.
- **PDF size cap raised to 50 MB** — PDF download and scrape payloads can now be up to 50 MB (up from 30 MB).
- **V1 aliases on V2 SDKs** — Python and JS V2 clients now expose deprecated `scrape_url` / `scrapeUrl`, `crawl_url` / `crawlUrl`, and friends so V1-trained callers keep working without rewriting code.
- **JS SDK string API key** — `new Firecrawl("fc-key")` now works alongside the object form, with whitespace-only keys rejected up front.
- **CVE patches** — Resolved `axios` (GHSA-35jp-ww65-95wh and friends, upgraded to `1.18.0`) across the API and JS SDKs, the Rust SDK's `openssl` (CVE-2026-41676), `@grpc/grpc-js` in the test-suite, `@opentelemetry/core` (GHSA-8988-4f7v-96qf), `@ai-sdk/provider-utils` (GHSA-866g-f22w-33x8), `esbuild` `0.28.1` (GHSA-gv7w-rqvm-qjhr, GHSA-g7r4-m6w7-qqqr), and pnpm-overrode `shell-quote`, `ws`, `brace-expansion`, `qs`, `uuid`, `js-cookie`, `form-data`, `js-yaml`, `protobufjs`, `vite`, and `@babel/core` across the API, JS SDK, Playwright service, test-suite, and ingestion UI.
